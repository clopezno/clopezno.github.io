GII 19.05 Mejora y nuevas funcionalidades de comunicación de una interfaz para planta piloto Este proyecto es la continuación del realizado en 2019 “Desarrollo de una interfaz para planta piloto”. En el laboratorio del área de Ingeniería de Sistemas y Automática disponemos de una serie de plantas piloto (plantas de control de temperatura y caudal de aire) conectadas y controladas mediante un sistema embebido de Freescale (placa FRDMK64F) y en el proyecto anterior se desarrolló una interfaz para PC en Windows que actúa como un SCADA (interfaz hombre máquina) para monitorizar variables del proceso así como actuar sobre él. El objetivo del actual TFG es ampliar las funcionalidades de la interfaz previamente desarrollada: •Añadir nuevas funcionalidades de comunicación entre la interfaz en el PC y la placa: comunicación TCP/IP, comunicación vía OPC (OPC es un estándar de comunicaciones industriales) y comunicación WIFI. •Actualmente la información del proceso (valor de las variables a lo largo del tiempo) se almacena en una base de datos local en Microsoft SQL. Se quiere sustituir esta base de datos por una más ligera, libre y sencilla de instalar, configurar y mantener, dado que la información que se almacena es muy poca. •Mejorar la interfaz visual. Trabajos relacionados: “Desarrollo de una interfaz para planta piloto”. Proyecto fin de carrera del grado de Ingeniería Informática. Universidad de burgos. GII 19.06 Implementación de estándares de simulación distribuida y co-simulación El trabajo consistirá en el estudio y desarrollo de pequeños ejemplos de aplicación de dos estándares de comunicación MPI (Message Passing Interface), que es un estándar que permite la comunicación entre procesos y que permite la ejecución concurrente de aplicaciones mediante el paso de mensajes. Se estudiaría el estándar y se utilizaría alguna de sus implementaciones en C/C++ en un ejemplo en que se comuniquen varias aplicaciones sencillas entre sí, para después aplicarlo a la comunicación entre simulaciones. El segundo de los estándares es el estándar FMI que permite la co-simulación de modelos dinámicos, permitiendo también la integración de modelos desarrollados en distintas herramientas. Se haría un estudio del estándar y se desarrollaría un pequeño ejemplo de aplicación del estándar. GII 19.07 Conectividad Wifi con sistemas embebidos En este proyecto se estudiarán las posibilidades de comunicación entre sistemas embebidos, en concreto la placa FRDM K64F y equipos PC portátiles / móviles. En una primera fase se añadirá un Shield wifi al sistema embebido para dotarlo de conectividad wifi siendo necesario programar y configurar el sistema embebido para que funcione como punto de acceso wifi y sea capaz de crear una WLAN o como sistema terminal wifi. Se probará la conectividad entre varias placas, portátiles y móviles. En una segunda fase se incluirá en los sistemas embebidos una implementación ligera del protocolo TCP/IP que sea capaz de adquirir una dirección IP utilizando DHCP. Se construirá una red LAN cableada a través de switches de varias placas y la misma red pero inalámbrica y se probarán distintas configuraciones como que los sistemas terminales respondan a solicitudes de ping, que ejecuten el protocolo ARP. El servidor DHCP se puede configurar y usar en Linux en un PC portátil o se puede usar un router WIFI doméstico que lo implemente. Una vez hecho esto deberá ser posible interactuar con un equipo portátil o móvil a través de wifi, de forma que seamos capaces de transmitir órdenes a las placas de manera remota y que las placas se comuniquen entre ellas. Trabajos relacionados: “Comunicación TCP/IP con sistemas empotrados”. Proyecto fin de máster del Máster Universitario en Ingeniería Informática. Universidad de burgos. Curso 2018-2019. GII 20.16 Generación automática de cuestionarios Moodle para asignaturas de inteligencia artificial Moodle es uno de los LMS (Learning Management Systems) más utilizado en la actualidad. Entre otras características ofrece la posibilidad de crear cuestionarios con los que evaluar el conocimiento adquirido por los alumnos. Desafortunadamente la interfaz para crear las preguntas es muy pesada y se hace difícil de usar cuanto se quieren crear preguntas de tipo cloze (preguntas con múltiples campos de respuesta). La propia gestión de las preguntas en el banco de preguntas de Moodle es mejorable. En este proyecto se pretende mejorar estos aspectos. En concreto centrándonos en lo que sería la generación automática de preguntas cloze sobre contenidos de asignaturas de inteligencia artificial: algoritmos de búsqueda, clasificadores, procesos de decisión de Markov, procesamiento de imágenes, lógica, ... El proyecto consistirá en el estudio e implementación de las técnicas conocidas como Edge Computing (para incluir procesamiento inteligente en el propio dispositivo) que permiten las bibliotecas de Tensorflow Lite, PyTorch Mobile o Matlab Mobile, para realizar las pruebas con datos en entornos reales y seleccionar el software más adecuado. Una vez seleccionado se trabajará en la implementación de un sistema embebido en el dispositivo correspondiente, sin necesidad de procesado de datos de forma externa. El resultado final esperado es un prototipo funcional de un sistema de control de aforo en alguno de los dispositivos mencionados. Bruno Baruque Zanón
