GII_O_MA_19.03 Implementación de estándares de simulación distribuida y co-simulación

Descripción

El trabajo consistirá en el estudio y desarrollo de pequeños ejemplos de aplicación de dos estándares de comunicación MPI (Message Passing Interface), que es un estándar que permite la comunicación entre procesos y que permite la ejecución concurrente de aplicaciones mediante el paso de mensajes. Se estudiaría el estándar y se utilizaría alguna de sus implementaciones en C/C++ en un ejemplo en que se comuniquen varias aplicaciones sencillas entre sí, para después aplicarlo a la comunicación entre simulaciones. El segundo de los estándares es el estándar FMI que permite la co-simulación de modelos dinámicos, permitiendo también la integración de modelos desarrollados en distintas herramientas. Se haría un estudio del estándar y se desarrollaría un pequeño ejemplo de aplicación del estándar.

GII_O_MC_19.06 Implementación de APIs para estándar XS2A

Descripción

PSD2 (Payment Services Directive) es una directiva europea que regula servicios de pago (transferencias, domiciliaciones, pagos con tarjetas, etc.) realizados en Europa, y entró en vigor el 14 de septiembre de 2019 en España. Basándose en PSD2 y en EBA RTS (European Banking Authority Regulatory Technical Standards) la iniciativa de estandarización 'Berlin Group' ha desarrollado el estándar XS2A (Access to Account Framework) que facilita la creación de productos y servicios innovadores (‘Banking as a Service’) mediante un conjunto de APIs de acceso a cuentas bancarias y datos financieros. Para la implementación de las APIs se utilizará una arquitectura basada un stack tecnológico Java 8 y Spring framework, así como elementos comunes a un entorno basado en APIs: API Gateway, API Portal utilizando el producto open source WSO2 API MANAGER. En el desarrollo se emplean buenas prácticas de ingeniería del software: Source control Métricas de calidad de código Continous integration / Continous Deployment Testing Task tracking SCRUM

GII_O_MA_19.07 Comparador de métricas de evolución en repositorios Software

Descripción

A partir de un Trabajo Fin de Grado se desarrolló una aplicación Web que recibe como entrada la dirección Web de un repositorio GitLab y calcula las medidas de evolución, genera un informe con la cobertura de medidas de evolución que están dentro de los valores umbrales para inferir una medidad orientativa del proceso. En este nuevo TFG se pretende continuar con el desarrollo original ampliando la funcionalidad, para permitir obtener datos tanto de Github como de GitLab, para permitir evaluar los forks de repositorios, para añadir un servicio de autenticación de usuarios y para mejorar el proceso de obtención de métricas de evolución con filtros temporales (mejora contínua). (Funcionalidad opcional a decidir por el estudiante) Utilizando las medidas de evolución de otros proyectos modelar un clasificador que permita estimar la evolución futura.

GII_O_MA_19.08 Desarrollo de un Sistema de Planificación y Gestión de la Contratación Pública

Descripción

La Secretaría General de Administración Digital (SGAD), con rango de Subsecretaría, y dependiente del Ministerio de Política Territorial y Función Pública es el órgano encargado de impulsar el proceso de racionalización de las tecnologías de la información y de las comunicaciones en el ámbito de la Administración General del Estado y sus Organismos Públicos. Dependiente de la SGAD se encuentra la Subdirección General de Inversiones TIC, que tiene asignada la gestión económica y financiera en el ámbito de la Secretaría General, y concretamente la planificación de las inversiones y gastos a realizar, así como la gestión de la contratación. El Trabajo Fin de Grado (TFG) propuesto deberá desarrollar un Sistema que permita la gestión la planificación y gestión de los expedientes por parte de la Subdirección General de Inversiones TIC, facilitando el proceso de contratación de la SGAD y permitiendo el intercambio de información con el resto de las Subdirecciones Generales de la SGAD.

GII_O_MA_21.02 Migración de Generador de Informes Nutricionales a plataforma Web

Descripción

Migración a plataforma web de una aplicación cliente para la generación de informes nutricionales sobre menús. Se plantea su implantación real en las instalaciones de restauración de los distintos centros de la Universidad de Burgos. La aplicación en web debe replicar los requisitos ya implementados en la aplicación cliente (ya existente) y adicionalmente permitir configurar menús para distintos proveedores. Se dará especial importancia a la seguridad en el almacenamiento de datos y recuperación de los mismos en caso de incidencias. La herramienta utiliza en la actualidad información nutricional de ingredientes extraída con webscraping, para la generación final de los informes nutricionales. Se utiliza el concepto teórico de Nutriscore. Técnicamente, frente a la solución actual con JavaFX se propone una migración a framework Vaadin (o Spring en su defecto). El despligeue debe prepararse sobre entorno GNU/Linux, aunque el desarrollo puede realizarse en cualquier plataforma (Windows, GNU/Linux o MacOS).

GII_O_MA_21.06 Aplicación web para la monitorización y el control de una planta piloto de laboratorio

Descripción

Se propone en este proyecto el desarrollo de un entorno web, que permita acceder de manera local o remota a una planta de laboratorio controlada por un sistema embebido. Este sistema empotrado es el encargado de controlar el proceso y de comunicarse con el PC. Se propone programar en el sistema embebido la comunicación a través de TCP/IP, utilizando el puerto ethernet del que dispone la placa o mediante una tarjeta wifi conectada a la placa, para enviar los datos obtenidos de la planta a un equipo en el que se podrá acceder a través de una interfaz web a los datos de la planta, enviando a su vez las consignas de operación necesarias. La interfaz permitirá además mostrar gráficamente algunas variables de proceso.
