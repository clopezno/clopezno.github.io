GII_O_MA_19.02Configuración automática de redes en GNS3

Descripción

Este proyecto es la continuación del proyecto realizado en el curso 2018/2019 denominado: “Simulación de redes con GNS3”. En este proyecto se simuló una red institucional con distintas funcionalidades utilizando el emulador de redes GNS3. Se propone continuar con este proyecto, extendiendo la funcionalidad esbozada en el proyecto anterior que permitía la configuración remota de los dispositivos de una red. Para ello, será necesario leer la topología de una red a partir del fichero de configuración de GNS3 y a partir de la misma, mediante un fichero de texto, por línea de comandos o mediante una pequeña aplicación de escritorio, será posible modificar la configuración de red de los dispositivos de la red detectados.

GII_O_MA_19.03 Implementación de estándares de simulación distribuida y co-simulación

Descripción

El trabajo consistirá en el estudio y desarrollo de pequeños ejemplos de aplicación de dos estándares de comunicación MPI (Message Passing Interface), que es un estándar que permite la comunicación entre procesos y que permite la ejecución concurrente de aplicaciones mediante el paso de mensajes. Se estudiaría el estándar y se utilizaría alguna de sus implementaciones en C/C++ en un ejemplo en que se comuniquen varias aplicaciones sencillas entre sí, para después aplicarlo a la comunicación entre simulaciones. El segundo de los estándares es el estándar FMI que permite la co-simulación de modelos dinámicos, permitiendo también la integración de modelos desarrollados en distintas herramientas. Se haría un estudio del estándar y se desarrollaría un pequeño ejemplo de aplicación del estándar.

GII_O_MC_19.06 Implementación de APIs para estándar XS2A

Descripción

PSD2 (Payment Services Directive) es una directiva europea que regula servicios de pago (transferencias, domiciliaciones, pagos con tarjetas, etc.) realizados en Europa, y entró en vigor el 14 de septiembre de 2019 en España. Basándose en PSD2 y en EBA RTS (European Banking Authority Regulatory Technical Standards) la iniciativa de estandarización 'Berlin Group' ha desarrollado el estándar XS2A (Access to Account Framework) que facilita la creación de productos y servicios innovadores (‘Banking as a Service’) mediante un conjunto de APIs de acceso a cuentas bancarias y datos financieros. Para la implementación de las APIs se utilizará una arquitectura basada un stack tecnológico Java 8 y Spring framework, así como elementos comunes a un entorno basado en APIs: API Gateway, API Portal utilizando el producto open source WSO2 API MANAGER. En el desarrollo se emplean buenas prácticas de ingeniería del software: Source control Métricas de calidad de código Continous integration / Continous Deployment Testing Task tracking SCRUM

GII_O_MA_19.07 Comparador de métricas de evolución en repositorios Software

Descripción

A partir de un Trabajo Fin de Grado se desarrolló una aplicación Web que recibe como entrada la dirección Web de un repositorio GitLab y calcula las medidas de evolución, genera un informe con la cobertura de medidas de evolución que están dentro de los valores umbrales para inferir una medidad orientativa del proceso. En este nuevo TFG se pretende continuar con el desarrollo original ampliando la funcionalidad, para permitir obtener datos tanto de Github como de GitLab, para permitir evaluar los forks de repositorios, para añadir un servicio de autenticación de usuarios y para mejorar el proceso de obtención de métricas de evolución con filtros temporales (mejora contínua). (Funcionalidad opcional a decidir por el estudiante) Utilizando las medidas de evolución de otros proyectos modelar un clasificador que permita estimar la evolución futura.

GII_O_MA_19.08 Desarrollo de un Sistema de Planificación y Gestión de la Contratación Pública

Dependiente de la SGAD se encuentra la Subdirección General de Inversiones TIC, que tiene asignada la gestión económica y financiera en el ámbito de la Secretaría General, y concretamente la planificación de las inversiones y gastos a realizar, así como la gestión de la contratación. El Trabajo Fin de Grado (TFG) propuesto deberá desarrollar un Sistema que permita la gestión la planificación y gestión de los expedientes por parte de la Subdirección General de Inversiones TIC, facilitando el proceso de contratación de la SGAD y permitiendo el intercambio de información con el resto de las Subdirecciones Generales de la SGAD.

GII_O_MA_19.0B Aplicación de técnicas metaheurísticas para la optimización del reparto de material de una planta de montaje

Descripción

Se quiere optimizar el proceso de reparto de material en una fábrica. El proyecto parte del análisis de la situación de una empresa internacional que actualmente produce neumáticos en Burgos, si bien, se pretende proponer una herramienta de optimización general que pueda ser utilizada en un mayor número de supuestos. El objetivo del trabajo es optimizar la entrega de los materiales que requieren un conjunto de máquinas y puestos de trabajo. En muchas ocasiones la falta de material provoca la parada de la máquina, con las consecuentes pérdidas en órdenes de fabricación no satisfechas y paradas de máquinas. En este trabajo, además concurren otro tipo de incidencias que pueden complicar esta asignación, por ejemplo, falta de material en el almacén, falta de espacio en la máquina para dejar el material, errores en el inventariado del almacén, errores en las ubicaciones del almacén, etc.

Trabajos teóricos relacionados

N. Xiong, D. Molina, M. Leon-Ortiz, F. Herrera. A walk into Metahueristics for Engineering Optimization: Principles, Methods and Recent Trends. International Journal of Computational Intelligent Systems (IJCIS), 8, 2015, 606-636.

GII_O_MB_19.0C ExoplanetIA La misión Kepler se diseñó para estudiar las estrellas del entorno cercano a la Tierra dentro de la Vía Láctea, analizando la luz emitida por 150.000 estrellas, para tratar de descubrir planetas de tamaño similar a la Tierra que orbitasen a su estrella dentro de la zona de habitabilidad, allí donde es posible que exista agua líquida. Con esta información se podría hacer una búsqueda de vida extraterrestre dirigida, así como una estimación de planetas potencialmente habitables. El objetivo del proyecto es utilizar técnicas de Machine Learning sobre los datos reales ofrecidos por el satélite espacial Kepler, con respecto a la intensidad de luz (flux) registrada durante más de 9 años, y proponer un modelo de red neuronal para detectar exoplanetas por el Método del Tránsito (https://hpscds.com/observatorio/#ExoplanetIA). Python y C++. Carlos López Nozal (Mención Ingeniería del Software)
