GII_O_MA_19.02Configuración automática de redes en GNS3 Este proyecto es la continuación del proyecto realizado en el curso 2018/2019 denominado: “Simulación de redes con GNS3”. GII_O_MA_19.03 Implementación de estándares de simulación distribuida y co-simulación El trabajo consistirá en el estudio y desarrollo de pequeños ejemplos de aplicación de dos estándares de comunicación MPI (Message Passing Interface), que es un estándar que permite la comunicación entre procesos y que permite la ejecución concurrente de aplicaciones mediante el paso de mensajes. Se estudiaría el estándar y se utilizaría alguna de sus implementaciones en C/C++ en un ejemplo en que se comuniquen varias aplicaciones sencillas entre sí, para después aplicarlo a la comunicación entre simulaciones. El segundo de los estándares es el estándar FMI que permite la co-simulación de modelos dinámicos, permitiendo también la integración de modelos desarrollados en distintas herramientas. Se haría un estudio del estándar y se desarrollaría un pequeño ejemplo de aplicación del estándar. **** Trabajos teóricos relacionados: https://www.mpi-forum.org/docs/ **** Trabajos prácticos relacionados: https://www.ecosimpro.com/exporting-models-using-fmi-standard/ https://computing.llnl.gov/tutorials/mpi/ Alejandro Merino Gómez (Mención en Ingeniería de Computadores)

GII_O_MC_19.06 Implementación de APIs para estándar XS2A PSD2 (Payment Services Directive) es una directiva europea que regula servicios de pago (transferencias, domiciliaciones, pagos con tarjetas, etc.) realizados en Europa, y entró en vigor el 14 de septiembre de 2019 en España. Basándose en PSD2 y en EBA RTS (European Banking Authority Regulatory Technical Standards) la iniciativa de estandarización 'Berlin Group' ha desarrollado el estándar XS2A (Access to Account Framework) que facilita la creación de productos y servicios innovadores (‘Banking as a Service’) mediante un conjunto de APIs de acceso a cuentas bancarias y datos financieros. Para la implementación de las APIs se utilizará una arquitectura basada un stack tecnológico Java 8 y Spring framework, así como elementos comunes a un entorno basado en APIs: API Gateway, API Portal utilizando el producto open source WSO2 API MANAGER. GII_O_MA_19.07 Comparador de métricas de evolución en repositorios Software A partir de un Trabajo Fin de Grado previo, se desarrolló una aplicación Web que recibe como entrada la dirección Web de un repositorio GitLab y calcula las medidas de evolución, genera un informe con la cobertura de medidas de evolución para inferir una medida orientativa del proceso. En este nuevo TFG se pretende continuar con el desarrollo original ampliando la funcionalidad, para permitir obtener datos tanto de Github como de GitLab, para permitir evaluar los forks de repositorios, para añadir un servicio de autenticación de usuarios y para mejorar el proceso de obtención de métricas de evolución con filtros temporales (mejora contínua). (Funcionalidad opcional a decidir por el estudiante) Utilizando las medidas de evolución de otros proyectos modelar un clasificador que permita estimar la evolución futura. **** Trabajos teóricos relacionados: - J. Appleton, Software Configuration Management Patterns: Effective Teamwork, Practical Integration, Edición: 01.
