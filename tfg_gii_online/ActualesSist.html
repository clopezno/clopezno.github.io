GII_O_17.01 Desarrollo de una interfaz para planta piloto Es un proyecto de desarrollo de software y de comunicaciones entre un pc/portátil y una placa. Tenemos en el laboratorio placas Freescale (FRDMK64F) conectadas a plantas piloto (plantas de control de temperatura y caudal de aire) y la idea es desarrollar una interfaz que actúe como un SCADA (interfaz hombre máquina) para monitorizar variables del proceso así como actuar sobre el proceso y un módulo de comunicaciones entre el PC y la placa. La interfaz hombre máquina iría implementada en el PC y habría que dotarla de un módulo de comunicaciones implementado tanto en el PC como en la placa que pueda soportar alguna de estas comunicaciones: wifi, Ethernet, comunicación serie, etc. El módulo de comunicaciones implementado en la placa será en C y la interfaz hombre máquina podrá ser en cualquier lenguaje, C++, visual basic, etc. Alejandro Merino Gómez

GII_O_MB_18.03 EnergySaver: Centralizar el encendido y apagado de equipos de forma remota utilizando un sitema de reglas. Modalidad B. El proyecto consiste en el desarrollo de una aplicación cliente-servidor en la que se puede programar el encendido y apagado de los sistemas que ejecutan el cliente utilizando un sistema de reglas. Tras el encendido, se podrán lanzar un conjunto de comandos o aplicaciones seleccionables por el usuario. El apagado se podrá controlar con reglas del tipo: "XXX min. tras la finalización del proceso YYYY y del proceso ZZZZ". Los clientes podrán ejecutarse en distintas plataformas (al menos Windows y Linux). Tecnologías a utilizar: Sistemas Operativos, Linux, Windows, C++, HTML, CSS Conocimientos a aprender en el desarrollo del proyecto: Sistemas operativos, gestión remota de equipos. **** Trabajos teóricos relacionados: A especificar por empresa HP SCDS. **** Trabajos prácticos relacionados: Carlos López Nozal (Mención ingeniería del Software)
