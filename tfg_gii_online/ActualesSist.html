GII_O_MC_18.09 Automatización del proceso de delineación a partir de datos de un levantamiento topográfico Motivación El trabajo que realizamos habitualmente los Ingenieros Topógrafos consta de dos partes fundamentales; trabajo de campo y gabinete. Un problema importante al que nos enfrentamos es la gran cantidad de tiempo y recursos que invertimos en gestionar los datos obtenidos en campo para obtener un plano del levantamiento topográfico ejecutado, alargando en exceso el desarrollo y entrega de un proyecto. Con este planteamiento surge la necesidad de crear una aplicación que consiga automatizar este trabajo el máximo posible y así obtener un ahorro de tiempo y de recursos, fundamental en nuestros proyectos. Puede parecer a priori un trabajo sencillo y rápido de hacer con una herramienta de cad, uniendo puntos manualmente para crear líneas, u otros elementos, claro está, si tienes un levantamiento topográfico con 30 puntos y recuerdas como lo has hecho en campo, pero cuando tienes un levantamiento con 10000 puntos, e intervienen varias personas en él, se puede convertir en un trabajo muy complicado e insufrible de realizar. Una herramienta que incluya las funcionalidades que aquí se pretenden implementar, no existe como tal en el mercado, algunos programas como Autocad o programas específicos de Topografía, disponen de algún módulo a mayores del programa, que permiten hacer algo similar, pero con un resultado muy pobre respecto a lo que aquí se plantea y necesitas un conocimiento avanzado de estos para conseguirlo. Objetivo Crear una aplicación que lea los datos extraídos del equipo topográfico, ya sea estación total o GPS y produzca un archivo de tipo dxf, compatible con los programas de cad (Autocad, MicroStation, DraftSight, etc), interpretando toda la codificación, separando los elementos en capas, dibujando todos los elementos medidos en campo, como líneas, curvas, símbolos, etc.. todo ello de forma automática. Hay que tener en cuenta que en el campo solo se miden puntos individuales con coordenadas x, y,z,todo el proceso para crear líneas rectas, curvas, superficies, etc, lo realizaría la aplicación interpretando la codificación. La aplicación no va a tener la pretensión de ser una aplicación de cad, ni nada parecido, sino una aplicación que automatice el proceso de interpretación de los datos provenientes de un levantamiento topográfico y genere una salida cad en formato dxf, automatizando lo máximo posible todo el proceso de dibujo e incluso pudiendo personalizar cada usuario la codificación en campo mediante librerías personalizadas. Herramientas: La aplicación se desarrollará con Python y Flask , usando las librerías de Python: Ezdxf,sdxf,Dxfgrabber **** Trabajos teóricos relacionados: https://www.delicad.com/en/rapiddxf.php https://www.delicad.com/en/rapiddxf_lt.php **** Trabajos prácticos relacionados: Carlos López Nozal

GII_O_MA_19.01Autoevaluación proceso de desarrollo de Trabajo Fin de Grado en Github Desarrollar una aplicación que reciba como entrada la dirección Web de un repositorio Github, calcule las medidas de evolución, genere un informe con la cobertura de medidas de evolución que están dentro de los valores umbrales para inferir una calificación orientativa del proceso. Con el conjunto de datos de las medidas de evolución se realizará un análisis estadístico para calcular los valores umbrales de las mismas (percentil 25 y percentil 75).
