CodeHL

CODEHL o Code HighLighter es un plugin desarrollado bajo JavaScript que se integra con el editor TinyMCE para proporcionar soporte al trabajo con código de distintos lenguajes de programación. Funciona con las distintas versiones de dicho editor y permite su integración con Moodle.

Este proyecto surge bajo la necesidad de crear preguntas incrustadas (CLOZE) en código formateado, por lo que en la descarga se incluyen dos plugins distintos e independientes. Usted es libre de instalar solo aquellos que desee.

  • CodeHL, compuesto por:
    • CodeHL: plugin para el formateo de código de distintos lenguajes de programación.
    • Cloze Export: plugin para la exportación de preguntas CLOZE. Permite la visualización e interpretación de preguntas CLOZE.
  • CLOZE Editor: plugin independiente de este desarrollo para la creación de preguntas CLOZE.
    Aviso: CLOZE Editor es un plugin pensado para su integración con Moodle. La versión incluida junto a CodeHL en sus versiones sin Moodle es una adaptación del plugin original para su funcionamiento externo, por lo que no se ofrece soporte por nuestra parte para este añadido. Para optener CLOZE Editor para Moodle visite su página oficial.

Instalación

Los requisitos mínimos para la instalación son:

  • TinyMCE: el editor TinyMCE es la base para el funcionamiento de los plugins. Para descargar e instalar el editor visite la página del editor.
    Aviso: TinyMCE requiere de un servicio Web para funcionar. Su uso de forma local inpide el correcto funcionamiento del editor y de los plugins.
  • Navegador Web con JavaScript habilitado.

Instalación en TinyMCE

  1. Descomprima el archivo descargado dentro de ruta_de_su_editor/tiny_mce/plugins/
  2. En la declaración del editor incluya lo siguiente:
    • En el listado de plugins del editor incluya el nombre del plugin: (codehl para CodeHL y Cloze Export, clozeeditor para CLOZE Editor). Elija lo que desee incluir.
    • En la lista de botones (Toolbar) indique los plugins a incluir en el orden deseado: codehl para CodeHL, clozeexport para CLOZE Export y clozeeditor para CLOZE Editor.

El resultado final debería ser algo similar a:

<script> 
tinymce.init({ 
    selector: "textarea#elm1", 
    theme: "modern", 
    width: 300, 
    height: 300, 
    plugins: [ 
         "codehl clozeeditor advlist autolink link image lists charmap print preview hr anchor pagebreak", 
         "spellchecker searchreplace wordcount visualblocks visualchars code fullscreen insertdatetime media nonbreaking", 
         "save table contextmenu directionality emoticons template paste textcolor" 
   ], 
   content_css: "css/content.css", 
   toolbar: "codehl clozeexport clozeeditor | insertfile undo redo | styleselect | bold italic | alignleft aligncenter alignright alignjustify |
                 bullist numlist outdent indent | ink image | print preview media fullpage | forecolor 
                 backcolor emoticons", 
   style_formats: [ 
        {title: 'Bold text', inline: 'b'}, 
        {title: 'Red text', inline: 'span', styles: {color: '#ff0000'}}, 
        {title: 'Red header', block: 'h1', styles: {color: '#ff0000'}}, 
        {title: 'Example 1', inline: 'span', classes: 'example1'}, 
        {title: 'Example 2', inline: 'span', classes: 'example2'}, 
        {title: 'Table styles'}, 
        {title: 'Table row 1', selector: 'tr', classes: 'tablerow1'} 
    ] 
 }); 
</script> 
 
 

Como parámetro adicional podemos configurar el atributo content_css para incluir la hoja de estilos del código, de tal forma que el resaltado de sisntaxis sea visible tras terminar el formateo. Si no se incluye esta opción el resultado generado será el mismo en ambos casos.

 content_css: "ruta_del_editor/tinymce/plugins/codehl/js/prettify/prettify.css",

Instalación en Moodle

El editor por defecto de Moodle desde su versión 2.0 a la versión 2.6 es TinyMCE en sus dos versiones (3 y 4). En la versión 2.7 TinyMCE no es la opción por defecto, pero sigue incluido en la distribución.

Para la instalación en Moodle, CLOZE Editor no está incluido en este descargable. Para conseguirlo visite su página oficial.

Los pasos para la instalación son similares, aunque varían en función de la versión de Moodle:


Moodle 2.0 a 2.3

  1. Descomprima el archivo descargado dentro de lib/editor/tinymce/
    Aviso: No sustituya el archivo version.php, ya que de lo contrario su distribución de Moodle se verá afectada.
    Aviso: El archivo lib.php contiene toda la configuración de su editor. Cree una copia de seguridad del mismo antes de remplazarlo, o añada la configuración de forma manual para conservar sus preferencias.
  2. Incluya el contenido del archivo lang/idioma/tinymce_codehl.php en el archivo lang/idioma/editor_tinymce.php

Moodle 2.4 o superior

  1. Descomprima el fichero en lib/editor/tinymce/plugins.
  2. Dirijase a 'Administratión > Site Administration > Notifications '
  3. Finalize la instalación pulsando el botón 'Upgrade Moodle database now'.

Moodle 2.5 o superior

  1. Descargue el paquete pero no lo descomprima.
  2. En el panel de Administración dirijase a 'Plugins/Install add-ons'
  3. Seleccione 'Install add-on from ZIP file'
  4. Seleccione 'TinyMCE HTML editor / tinymce (tinymce)' como tipo de plugin.
  5. Navegue hasta la ruta donde descargó el fichero y selecciónelo.
  6. Acepte los términos de uso y siga las instrucciónes.
  7. Finalizar la instalación pulsando el botón 'Actualizar la Base de Datos de Moodle ahora'.

Ayuda

Ejecución en local / El editor no funciona al hacer doble click en el archivo HTML.

El editor TinyMCE necesita de un servicio Web para funcionar. En caso contrario, las políticas de seguridad de su navegador Web impedirán la ejecución de los plugins.

Truco: en Google Chrome se pueden desactivar los bloqueos de seguridad ejecutando el navegador con las opciones –allow-file-access-from-files –disable-web-security. Esta es una opción NO recomendable, ya que deja la puerta abierta a posibles amenazas.
